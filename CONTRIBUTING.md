# Contributing to Web Search Agent

Thank you for your interest in contributing! This document provides guidelines for contributing to this project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/web-search-agent.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Commit: `git commit -m "Description of changes"`
7. Push: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add comments for complex logic
- Include docstrings for functions and classes

## Testing

Before submitting a PR:
- Test your changes with multiple queries
- Ensure benchmarking still works
- Check that error handling functions properly

## Areas for Contribution

- Performance optimizations
- New features (see Future Enhancements in README)
- Bug fixes
- Documentation improvements
- Test coverage
- UI/UX improvements

## Questions?

Feel free to open an issue for any questions or discussions!
